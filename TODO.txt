Wave Survival 0.4
-----------------

Bugfixes
--------
[游리] Fixed enemies sometimes spawning in the ground
[游리] Enemy units no longer try to attack bridges or pile up near them
[游리] Ranged units no longer pile up inside buildings
[仇] Fixed a bug where teams would randomly switch

Quality of Life
---------------
[游리] Deleting a completed building gives a 50% refund (scaled to how damaged it is)
[游리] Wave difficulty is advanced on each night regardless of if you cleared the previous wave or not
[游리] Waves no longer repeat the same faction two nights in a row
[游리] The spawn warning is now only shown if less than the expected number of units spawned
    - If you see one of these warnings, please show me a screenshot of your world
[游리] Added another debug option:
    - /debug-next-night - advances to the next night and wave

Piglin wave changes
-------------------
[游리] Reduced portal count from 1/2 to 1/3 of wave number but with the same total population each wave
[游리] Portals can no longer spawn over water (but they can still spawn on land across water)
[游리] Portals can no longer spawn too close to another portal
[游리] Portals no longer spawn units periodically if there's already too many enemies (2x current wave population)
[游리] Enemy ghasts no longer friendly fire their own portals

Features
--------
[游리] Improved pathfinding accuracy for all units
    - Increasing the accuracy also increased server CPU usage, so several optimisations have also been included:
        - If the last move command is identical to the current one (same units, same location, etc.) the command is ignored
        - When moving far distances (>50 blocks), some commands are ignored if the last move command was to a similar location
        - Move commands across water/lava are ignored if the unit would not move as a result (eg. they're stuck at the coast)

[仇] Configurable Unit Costs (Thanks to @DoodleNoodles)
    - Includes all costs, population values and build times
    - Modify the files generated at configs/reignofnether-config.toml to adjust them
    - If the server config is adjusted, this will be enforced for all players joining the match

Bugfixes

[游리] Spectators can no longer cancel units from RTS players' building queues
    - test in lilypad server?
[游리] Fixed some units randomly spawning as neutral
[九] Can no longer enter enemy garrisons

[九] Tall grass and barrier blocks are now ignored by the cursor

[九] Fix weird pathing over magma
    - To fix this, there is now a new block identical in texture and function to magma called "Walkable Magma"
    - @Mapmakers should replace any regular magma blocks in their maps with this block to improve pathfinding

[仇] Fix #localisation channel problems
[仇] Wipe all saved data on /rts-reset
[仇] You no longer have to restart your client to change button languages
